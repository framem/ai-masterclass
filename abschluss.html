Capstone

Eine Aufgabe, der gesamte Stoff

Der Selbst-Check zeigt, was du weißt. Der Capstone zeigt, was du kannst. Eine kleine, echte Aufgabe — durchgezogen vom Prompt bis zum Beleg, dass es funktioniert.

Die Mission Baue einen AI-Helfer für eine echte Aufgabe — und weise nach, dass er gut ist. Kein Tool-Wettbewerb, kein Production-System. Ziel ist die durchgängige Denkweise: einen Prompt entwerfen, das Vorgehen wählen, Leitplanken setzen — und am Ende mit einer Zahl auf festen Fällen zeigen, dass deine Lösung besser ist als der erste Entwurf. Wähle eine der zwei Spuren.
Spur A · Dev Commits → Release-Changelog Baue einen Helfer, der aus einer Liste von Commit-Messages einen sauberen, gruppierten Changelog macht — ohne erfundene Einträge, mit Ticket-Bezug. Alternativen: PR-Pre-Review · Test-Generator aus einer User-Story · Log-Triage.
Spur B · Fachbereich Mail-Eingang → Action-Items Baue einen Helfer, der aus einer langen E-Mail klare Action-Items + einen Antwortentwurf zieht — nichts erfindet, Fristen korrekt überträgt. Alternativen: Meeting-Notiz → To-dos · Angebots-Triage · FAQ-Antwortentwurf.

Fünf Liefergegenstände — je einer pro Kapitel. Kurz halten: ein Satz bis ein Absatz reicht.

Kapitel 01 Prompt & Context. Dein finaler Prompt (Rolle, Aufgabe, Format) plus: welcher Kontext ist immer dabei, welcher nur on-demand?
Kapitel 02 + 03 Vorgehen. Die EPEC-Zerlegung der Aufgabe — und die Entscheidung: fester Workflow oder Agent? Welche Tools/MCP, falls überhaupt?
Kapitel 04 Guardrails. Welche der vier Ebenen greifen bei dir? Nenne eine konkrete Bedrohung, die du aktiv mitigierst.
Kapitel 05 Eval. Ein Golden-Set von ≥ 5 echten Fällen, der gewählte Scorer, und der Score zweier Prompt-Varianten (A vs. B).
Kapitel 05 Trust-Entscheidung. Wo sitzt deine Aufgabe auf der Schaden×Prüfbarkeit-Matrix — und welche Prüf-Strategie folgt daraus?

Bewertungsraster

So wird ein guter Capstone von einem schwachen unterschieden — Maßstab ist die Nachweisbarkeit, nicht die Politur.

KriteriumSchwachStark
Prompt & Context Ein langer Wunschsatz; gesamter Kontext immer geladen. Strukturiert; bewusst getrennt, was immer vs. on-demand geladen wird.
Vorgehen Modell direkt angefragt, kein erkennbares Vorgehen. Begründete Wahl Workflow vs. Agent; EPEC sichtbar.
Guardrails Keine Guardrails benannt. Mind. eine Ebene + eine konkret mitigierte Bedrohung.
Evaluation Sichtprüfung, keine Testfälle. Golden-Set + Scorer + Zahl für A und B.
Trust Vertrauen ohne Begründung. Verortet auf der Matrix; Strategie folgt aus Schaden & Prüfbarkeit.
Format
Paare oder Trios. Eine Folie / ein Doc pro Liefergegenstand reicht.
Timebox
60–90 Min Bauen · 5 Min Vorstellen pro Gruppe.
Abgabe
Das Golden-Set + die zwei Scores sind das Herzstück — der Rest erklärt den Weg dahin.
Moderation Erst die Spur wählen lassen, dann die Aufgabe einfrieren. Die häufigste Falle ist Scope-Creep — wer ein Production-System baut, kommt nie zur Eval. Das Lernziel ist die Kette, nicht das Produkt.
