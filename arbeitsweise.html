Vorbereitung — Wie viel du dem Agenten überlässt

Befehl, Plan
oder Ziel.

Dieselbe Aufgabe, drei Eingriffstiefen. Je autonomer, desto mehr denkt der Agent selbst — und desto mehr Kontrolle gibst du an Freigabe-Punkte und Guardrails ab.

So klein wie möglich So autonom wie nötig

Stufe 01 — du steuerst
Befehl
„Du führst, das Modell führt aus — jeder Schritt unter deiner Kontrolle."
Eingriff: jeder Schritt Kontrolle maximal
Was
Eine konkrete Anweisung. Der Agent führt genau sie aus — ein Schritt, ein Ergebnis, sofort prüfbar.
Beispiel
du »Benenne getUser überall zu fetchUser um.«
agent → 14 Stellen in 6 Dateien umbenannt ✓
du »Füg hier einen Null-Check ein.«
agent → 3 Zeilen ergänzt ✓
Wann
Klein und eindeutig. Du weißt genau, was du willst — eine punktgenaue Änderung.
Grenze
Du denkst, der Agent tippt. Skaliert nicht auf große, mehrstufige Aufgaben.
Merksatz
Wenn du es in einem Satz präzise sagen kannst — sag es als Befehl. Kein Plan, kein Overhead.

Stufe 02 — du gibst den Plan frei
Plan
„Das Modell erstellt einen Plan, du gibst frei — die Umsetzung folgt erst nach deiner Freigabe."
Eingriff: Plan-Freigabe wichtigster Checkpoint
Was
Du beschreibst das Ergebnis. Der Agent schlägt zuerst einen Plan vor; du gibst frei oder korrigierst — dann erst die Umsetzung.
Beispiel
du »Pagination auf der Orders-Liste — erst ein Plan.«
agent → Plan:
1·Query auf LIMIT / OFFSET umstellen
2·Seiten-Controls in die UI einbauen
3·Tests für leere & letzte Seite
du → »Punkt 2 später — Rest freigegeben.«
Wann
Mittlere Aufgaben über mehrere Dateien; Unklarheit, die du vor dem Code sehen willst.
Gewinn
Fängt „die falsche Aufgabe gelöst" ab, bevor überhaupt etwas erstellt ist.
Merksatz
Einen Plan zu ändern ist günstig — fertigen Code umzuwerfen teuer. Lies den Plan, bevor du ihn freigibst.

Stufe 03 — du delegierst das Ziel
Ziel
„Du definierst das Ziel, der Agent liefert — du kontrollierst das Ergebnis."
Claude · Goal Copilot · Autopilot
Was
Du setzt Ziel + Erfolgskriterium — ohne Plan-Freigabe. Der Agent plant und fährt den ganzen Loop selbst; dein Eingriff erst am fertigen Ergebnis.
Beispiel
du »Mach die Test-Suite grün.«
agent → fixt 3 Tests, läuft 2× nach … ✓ grün
du »Migrier das Modul auf die neue API, öffne einen PR.«
agent → umgestellt, getestet, PR #248 offen
Wann
Klar abgegrenzte, aber mehrstufige Arbeit mit guten Guardrails (Tests, CI), denen du vertraust.
Risiko
Größter Handlungsspielraum. Braucht Tests / Sandbox; wandert bei unscharfem Ziel.
Merksatz
Autonomie ist nur so sicher wie deine Guardrails. Erst Tests, dann Autopilot.
