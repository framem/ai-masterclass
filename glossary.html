Wortliste.

Die wichtigsten Begriffe aus dem gesamten Workshop — kompakt zum Nachschlagen, von A2A bis Workflow.

38 von 38 Begriffen
A2A
Agent-to-Agent — offenes Protokoll, mit dem Agenten verschiedener Anbieter sich finden und Aufgaben austauschen können.
Agent
LLM, das selbst entscheidet, welche Werkzeuge es in welcher Reihenfolge benutzt, um ein Ziel zu erreichen.
Agent Card
Öffentliches Profil eines A2A-Agents: Fähigkeiten, Endpunkt, Authentifizierung — damit andere Agenten ihn finden.
Agentic Loop
Zyklus aus Denken und Handeln: der Agent plant, ruft ein Tool, beobachtet das Ergebnis und entscheidet neu — bis das Ziel erreicht ist. Gegenstück zur linearen Pipeline.
AGENTS.md
Projektweite Anweisungsdatei für Coding-Agenten: Tech-Stack, Konventionen, Standards. Wird als Kontext mitgeladen, damit der Agent im Haus-Stil arbeitet.
Autoregressiv
Erzeugungsweise eines LM: Es schreibt Token für Token — jedes neue Token hängt an allem bisher Geschriebenen. Kein Plan im Voraus, nur immer der nächste Schritt.
Context Engineering
Die Disziplin, dem Modell genau das Richtige in den Kontext zu legen — nicht möglichst viel. Während der Prompt das WAS liefert, liefert der Context das WORAUF.
Context Rot
Qualitätsverlust bei zu vollem Kontextfenster: das Modell verliert den Fokus, „vergisst" frühe Anweisungen und halluziniert eher.
Context Window
Maximale Token-Menge, die das Modell bei einer Anfrage gleichzeitig sehen kann (z. B. 200 K – 1 M).
Embedding
Übersetzung von Text in einen Punkt im hochdimensionalen Raum. Ähnliche Bedeutung → nahe Punkte.
Eval / Evaluation
Systematische Messung der Output-Qualität gegen Testfälle — statt Sichtprüfung nach Gefühl. Macht zwei Prompt-Varianten anhand einer Zahl vergleichbar.
Few-Shot
Dem Prompt ein paar Beispiele (Input → Output) mitgeben, statt die Aufgabe zu erklären. Drei Beispiele schlagen oft 500 Wörter Anleitung.
Golden-Set
Kuratierte Sammlung echter Fälle mit erwartetem Ergebnis — die Messlatte, gegen die jede Eval läuft.
Guardrail
Schutzschicht, die Ein- und Ausgaben eines KI-Systems prüft und begrenzt — z. B. PII-Filter, Prompt-Injection-Erkennung, Confidence-Schwelle. Wirkt in mehreren Ebenen.
Hallucination
Wenn das Modell etwas Plausibles, aber Falsches generiert. Folge daraus: Outputs gegen Quellen prüfen.
LLM-as-Judge
Ein zweites LLM bewertet den Output des ersten — nützlich, wenn es keine eine richtige Antwort gibt, sondern Qualität (Ton, Plausibilität) zählt.
Logit
Roh-Score, den das Modell jedem möglichen nächsten Token gibt. Wird mit Softmax in Wahrscheinlichkeiten umgewandelt.
MCP
Model Context Protocol — offener Standard, der Tools (Server) und Agenten (Clients) verbindet. USB-C für KI-Werkzeuge.
Memory
Persistenter Speicher über einzelne Gespräche hinaus — Fakten, Präferenzen, Historie. Anders als der Context, der pro Aufruf neu gefüllt wird.
Multi-Agent
Mehrere spezialisierte Agenten arbeiten zusammen — als Supervisor-Team, Pipeline oder Schwarm.
Post-Training
Phase nach dem Pretraining: RLHF & Präferenzdaten formen Ton, Hilfsbereitschaft, Verweigerungen und Tool-Use. Der größte im Alltag spürbare Unterschied zwischen Modellen mit ähnlichen Rohdaten.
Prompt
Die Eingabe an ein Modell — die „Bestellung". Sagt, was passieren soll (Aufgabe, Rolle, Format); der Context liefert das worauf.
Prompt Injection
Angriff, bei dem manipulierte Eingaben — oder Inhalte aus Tools/Web — das Modell zu unerwünschten Aktionen verleiten. Top-Bedrohung für Agenten.
RAG
Retrieval-Augmented Generation — eigene Dokumente per Vektor-Suche dem LLM mitgeben.
ReAct
Reason + Act — Muster für Agenten-Loops: abwechselnd nachdenken (Reason), handeln (Act) und das Ergebnis beobachten (Observe).
Reasoning Effort
Stellrad, wie viel ein Modell vor der Antwort „nachdenkt". Mehr ≠ besser: bei einfachen Aufgaben kostet es nur Zeit und Tokens (Overthinking).
Sampling
Dritter Schritt der LM-Pipeline (Tokenize · Predict · Sample): aus der Wahrscheinlichkeitsverteilung wird ein Token gezogen. Die Temperatur steuert, wie strikt der wahrscheinlichste Pfad gewählt wird.
Scorer
Die Funktion, die einen Output bewertet — von exakt (exact-match) über Regex bis LLM-as-Judge. Passend zur Aufgabe gewählt.
SDLC
Software Development Life Cycle — Plan, Code, Test, Review, Deploy, Operate. Zeigt, wo KI andockt und wo Begleitpflicht herrscht.
Skill
Wiederverwendbares Paket aus Anleitung + Werkzeugen, das ein Agent bei Bedarf lädt (SKILL.md) — nur dann im Kontext, wenn er gebraucht wird.
SLM / LLM
Small / Large Language Model. Klein, lokal, spezialisiert vs. groß, breit, im Rechenzentrum.
Softmax
Mathe-Funktion, die beliebige Zahlen in Prozente verwandelt, die zusammen 100 % ergeben.
System-Prompt
Persistente Anweisung, die bei jedem Aufruf vorab gesendet wird — Rolle, Ton, Regeln. Gilt fürs ganze Gespräch, anders als die einzelne User-Nachricht.
Temperatur
Stellrad für Zufall: 0 = immer höchster Logit · >1 = mehr Vielfalt, mehr Risiko.
Token
Kleinster Sprachbaustein für ein LLM — ein Wort, ein Wortteil oder ein Satzzeichen. „Brötchen" kann z. B. Brötch + en sein.
Tool
Funktion mit Name, Parametern und Rückgabewert, die ein Agent aufrufen darf — z. B. search_db, send_email.
Vektor-DB
Spezialdatenbank, die Embeddings speichert und blitzschnell die nächsten Nachbarn findet.
Workflow
Vorab definierte Kette von Schritten, in der das LLM nur an festen Stellen aufgerufen wird.
